Este es el quinto contrato inteligente de mi colección profesional "Sabores de América", desplegado en la red Base. Belice actúa como un puente cultural entre Centroamérica y el Caribe, ofreciendo una fusión única de influencias mayas, criollas y garífunas. 🛠️ Detalles Técnicos: Estructura y Metadatos

Este contrato evoluciona la arquitectura de la serie al añadir:

Identificador de Acompañamiento (string): Uso de la variable baseCarbohidrato para categorizar platos según su componente principal (Coco, Harina, Yuca o Maíz), esencial para la gastronomía beliceña.

Sistema de Votación Descentralizada: Funciones inmutables para darLike y darDislike que permiten medir el impacto global de estos sabores caribeños.

Eficiencia en Capa 2: Lógica optimizada para la red Base, manteniendo un límite de 200 caracteres por registro para asegurar transacciones económicas.

📖 Modo de Uso

Consultar Menú: En la pestaña Read Contract, usa consultarPlato (ID 1 para Rice and Beans) para ver la descripción y su base alimenticia.

Registrar Plato: En la pestaña Write Contract, usa registrarPlato para añadir nuevas especialidades (ej. "Fry Jacks", "Masa frita esponjosa en forma de triangulo", "Harina").

Votar: Califica los sabores de Belice directamente en la blockchain.
